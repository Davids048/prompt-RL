# Evaluation Repositories

Last updated: 2026-06-05

This file records the direct subdirectories under `evaluations/`, why they are present in this workspace, where they are used, and what local changes exist. The workspace root is not currently a git repository, but every direct `evaluations/` subdirectory is its own git checkout.

Related files:

```text
.agents/memory/image_calibration.md
  Run-level record of how GenEval, OCR, PickScore, HPS, and UnifiedReward were used.

.agents/memory/infrastructure.md
  Environment notes for `.venv`, `.envs/reward_server`, FastVideo, Slurm, and reward-server.

experiments/image_calibration/calibration_pipeline.py
  Main local pipeline that imports or calls the evaluator code.

experiments/image_calibration/run_promptrl_full_benchmarks.sh
  Full-run wrapper that chooses prompt sources and starts evaluator services.

evaluations/flow_grpo/README.md
  Upstream provenance for using `yifan123/reward-server` for GenEval reward preparation.

papers/related_works.md
  Research-side evaluator inventory and metric rationale.

papers/image_video_model_eval_report.md
  2026-06-05 synthesis of image/video generation papers and their eval stacks,
  based on `papers/evals.md`, related-work notes, local DanceGRPO findings, and
  paper/project sources. It maps image evals such as GenEval, OCR/MARIOEval,
  PickScore, HPS, UnifiedReward, and EditReward; video evals such as VBench,
  VideoReward/VideoAlign, VisionReward, VidProM, VideoJAM-bench, and MotionBench;
  and a high-level plan for adding the paper/eval matrix to Notion.
```

Reward-server provenance:

The GenEval `reward-server` was used because Flow-GRPO documents a separate reward-model server setup to avoid dependency conflicts across reward models. In `evaluations/flow_grpo/README.md`, section `3. Reward Preparation` -> `GenEval` says to create a new Conda environment and install the corresponding dependencies according to `reward-server` at `https://github.com/yifan123/reward-server`. This workspace followed that provenance chain by cloning `https://github.com/yifan123/reward-server.git` into `evaluations/reward-server` and running GenEval scoring through it, so the legacy `mmdet`/`mmcv`/`open_clip` stack stays out of the shared `.venv`.
